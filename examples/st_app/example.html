📋 Список возможностей фреймворка

  • Реактивность: автоматическое обновление DOM при изменении данных
  • Интерполяция: вывод данных через {{ expression }}
  • Двусторонняя привязка: атрибут #model для форм
  • События: обработчики через @event="handler()"
  • Условный рендеринг: #if, #else-if, #else
  • Циклы: #for="item in items" — для массивов доступны элемент и $index; для объектов — значение и $key
  • Видимость: #show="condition" для скрытия элементов
  • Вычисляемые свойства: геттеры с автоматическим пересчётом
  • Атрибуты с :: передача данных в дочерние компоненты; :attr без значения — значение из прототипа (в т.ч. JSON-объект)
  • Вложенные компоненты: <st-app> внутри <st-app>
  • innerHTML как шаблон: переопределение template через содержимое элемента
  • applyTemplate(): переключение шаблонов на лету; без аргументов — возврат к базовому
  • watch: this.watch(source, callback, options?), в колбэке unwatch() — третий аргумент

1. Счётчик (Базовая реактивность)

Демонстрация базовой реактивности, интерполяции и обработки событий.

2. Условный рендеринг (#if/#else-if/#else)

Показ/скрытие элементов на основе условий.

3. Списки и циклы (#for)

Директива #for: для массивов доступны элемент и $index; для объектов — значение и $key.

4. Формы и двусторонняя привязка (#model)

Связывание полей форм с данными через #model и вычисляемые свойства.

5. Todo List (Комплексный пример)

Приложение с несколькими вычисляемыми свойствами и методами.

5a. watch() и unwatch()

Отслеживание свойств, геттеров и вызовов методов. Остановка из колбэка через unwatch() (третий аргумент). Откройте консоль.

6. Вложенные реактивные объекты

Реактивность работает на любом уровне вложенности объектов.

7. Передача данных через атрибуты с :

Атрибуты, начинающиеся с :, парсятся как данные и передаются в компонент.

Примечание: Атрибуты :initial-count, :step и :label преобразуются в свойства initialCount, step и label компонента.

7a. Обратная совместимость: свойство — JSON-объект из прототипа

Если атрибут :attr указан без значения, значение берётся из конфига App.create(). Для объекта атрибут синхронизируется с JSON-строкой.

Примечание: Используется <st-app app="jsonProps" :options> — без значения у :options. Свойство options задано в прототипе объектом { theme, limit } и подтягивается в компонент; при изменении свойства атрибут обновляется.

8. Использование innerHTML как шаблона

Содержимое элемента <st-app> переопределяет шаблон из App.create().

Примечание: Приложение inlineTemplate объявлено с пустым template, но использует разметку внутри элемента.

Обновлено: {{ updateCount }} раз(а)

9. Вложенные компоненты (родитель → ребёнок)

Родительский компонент передаёт данные дочернему через атрибуты с :.

Примечание: Дочерний компонент получает данные через атрибуты :count и :name. После исправления бага #6, изменения в родителе автоматически синхронизируются с дочерним компонентом.

10. Вложенные компоненты (родитель → ребёнок → внук)

Трёхуровневая вложенность компонентов с передачей данных между уровнями.

11. Комбинированный пример (все возможности вместе)

Список элементов (цикл #for) — клик по пункту передаёт значение во вложенный <st-app> через атрибут :item; блок деталей рендерится из innerHTML дочернего компонента. Показаны: реактивность, #show, @click, вложенный компонент, передача данных через :, шаблон из innerHTML.

12. applyTemplate() — переключение между const-шаблонами

Cтартует с экрана загрузки. После задержки автоматически переключается на главный экран. Навигация между экранами через applyTemplate().

Примечание: applyTemplate() без аргументов всегда возвращается к базовому template из App.create(). applyTemplate(html) одноразово применяет переданную разметку, не перезаписывая базовый шаблон. Реактивные данные (name, theme, counter) сохраняются при любом переключении.
