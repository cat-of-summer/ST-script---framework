## 1. Описание

`Toggle` — универсальный примитив переключения состояния. Один экземпляр обслуживает
**множество независимых триггеров**; по событию (`click` и т.п.) триггер переключается между
состояниями и вместе с собой переводит свои **цели** (targets). Класс приходит на смену
`st_accordion` и достаточно общий, чтобы строить на нём аккордеоны, кнопки-переключатели,
спойлеры, hover-меню и т.п.

**Ключевые принципы:**

- **Только состояние.** Класс лишь меняет атрибут `state` на триггере и целях и вызывает
  пользовательские замыкания. Он **не трогает** `display`/`style` — весь визуал отдаётся CSS
  (по селекторам `[state="..."]`).
- **Нет встроенных групп.** Взаимоисключение (как у аккордеона) пишется вручную в хуке
  `on_activate`/`before_activate` через вызов `deactivate()` у соседей.
- **Триггеры и цели независимы.** Один target может управляться несколькими триггерами; источник
  направления — состояние самого триггера. События триггера и целей раздельные и асинхронные —
  это позволяет комбинировать цепочки toggle-ров (цель одного может быть триггером другого).
- **Нет зависимостей.** Чистый ES6-класс, экспортируется как ES-модуль.

## 2. Модель: триггеры и цели

```
new Toggle({ trigger: '[trigger]', target: null })
        │
        ├── trigger #1 ──► targets: [ ... ]   (из атрибута target="" или params.target)
        ├── trigger #2 ──► targets: [ ... ]
        └── trigger #N ──► targets: [ ... ]
```

- **Триггеры** ищутся один раз при создании по селектору `params.trigger` (по умолчанию
  `[trigger]`). На каждый вешается обработчик события `params.action`.
- **Цели** разрешаются **при каждом переключении** (а не один раз) — см. раздел
  [8](#8-разрешение-целей-target). У одного триггера может быть несколько целей.
- Каждый триггер хранит ссылку на экземпляр через приватный Symbol — доступно через
  `Toggle.find()`.

## 3. Машина состояний

Атрибут `state` (на триггере и на каждой цели) проходит цикл:

```
            activate                    on_activate / on_activated
inactive ──────────────► activating ──────────────────────────────► active
   ▲                                                                   │
   │  on_deactivate /            deactivate                            │
   └──── on_deactivated ◄──── deactivating ◄───────────────────────────┘
```

| Состояние | Описание |
|---|---|
| `inactive` | Выключено (конечное состояние после деактивации) |
| `activating` | Идёт анимация включения (прошёл один `requestAnimationFrame`) |
| `active` | Анимация завершена, включено |
| `deactivating` | Идёт анимация выключения |

### Как происходит переход (внутренняя механика)

Мирроринг механики `Modal`:

1. Решается направление (для `toggle()` — по текущему состоянию триггера).
2. Вызывается хук `before_*` (на старте).
3. Первый `requestAnimationFrame` ставит промежуточное состояние
   (`activating` / `deactivating`) — это запускает CSS-переход.
4. Второй `requestAnimationFrame` запускает `setTimeout(duration * 1000)`.
5. По окончании таймаута ставится финальное состояние (`active` / `inactive`) и
   вызывается хук `on_*`.

Триггер и его цели переводятся **отдельными** проходами (независимые жизненные циклы), но в
рамках одного прохода состояния всех целей меняются «пачкой» в одном `requestAnimationFrame` —
это снижает нагрузку при большом числе целей. Таймер хранится **per-element**, поэтому повторное
переключение прерывает ожидающий финал только нужных элементов.

### allow_interrupt

По умолчанию (`allow_interrupt: false`) переключение, вызванное во время обратной анимации,
игнорируется: `activate` работает только из `inactive`, `deactivate` — только из `active`.

При `allow_interrupt: true` разрешается прервать текущий переход и стартовать обратный
(`activate` из `deactivating`, `deactivate` из `activating`) — для плавного реверса.

## 4. Параметры конструктора

```js
new Toggle(params)
```

Все параметры необязательны.

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `trigger` | `string \| Element \| NodeList \| Array` | `'[trigger]'` | Что считать триггерами. Селектор резолвится через `querySelectorAll` один раз при создании. |
| `target` | `string \| function \| Element \| NodeList \| Array \| null` | `null` | Цели триггера. Если функция — вызывается как `target(trigger)` при каждом переключении. Если `null` — берётся из атрибута `target=""` триггера. См. раздел [8](#8-разрешение-целей-target). |
| `action` | `string` | `'click'` | Имя DOM-события на триггере. Спец-значение `'hover'` → `mouseenter` включает, `mouseleave` выключает. |
| `duration` | `number` | `0` | Длительность перехода в секундах (= время до установки финального состояния и вызова `on_*`). CSS-`transition` задаётся разработчиком. |
| `allow_interrupt` | `boolean` | `false` | Разрешить прерывание текущего перехода обратным. |
| `anchor` | `boolean` | `false` | Синхронизировать активный триггер с хешем URL (deep-linking). См. раздел [12](#12-якорные-ссылки-deep-linking). |
| `keep_history` | `boolean` | `false` | Создавать ли запись в истории на каждое переключение. `false` → `replaceState` (кнопка «назад» уходит со страницы); `true` → `pushState` + реакция на `hashchange` (кнопка «назад» возвращает предыдущий таб). **Принудительно включает `anchor: true`** (без якоря история хеша бессмысленна). |

> **Имена атрибутов фиксированы:** триггеры по умолчанию `[trigger]`, цель — атрибут `target=""`,
> состояние — атрибут `state=""`.

## 5. Хуки жизненного цикла

Хуки делятся на **события триггера** и **события целей** — они независимы.

### События триггера — вызываются 1 раз, аргументы `(trigger, targets)`

| Хук | Когда | `targets` |
|---|---|---|
| `before_activate` | на старте включения | массив целей |
| `on_activate` | после `duration` (включено) | массив целей |
| `before_deactivate` | на старте выключения | массив целей |
| `on_deactivate` | после `duration` (выключено) | массив целей |

### События целей — вызываются **для каждой цели**, аргументы `(target, trigger)`

| Хук | Когда |
|---|---|
| `before_activated` | на старте включения цели |
| `on_activated` | после `duration` (цель включена) |
| `before_deactivated` | на старте выключения цели |
| `on_deactivated` | после `duration` (цель выключена) |

### Инициализация

| Хук | Когда |
|---|---|
| `before_init` | до поиска триггеров и навешивания обработчиков; получает `params` |
| `on_init` | после полной инициализации; получает `params` |

### Привязка контекста

Все хуки (кроме `target`-функции) привязываются к экземпляру через `.bind(this)` — внутри хука
`this` указывает на экземпляр `Toggle`.

## 6. Публичные свойства и методы

### Геттеры

| Член | Возвращает | Описание |
|---|---|---|
| `params` | `object` | Параметры экземпляра (без функций-хуков) |

> Текущее состояние читается прямо из DOM: `el.getAttribute('state')` — отдельного метода нет.

### Методы

Все три принимают `Element`, `NodeList`, `Array` **или** CSS-селектор; применяются к каждому
совпавшему элементу.

#### `toggle(el)`

Переключает направление по текущему состоянию триггера: из `active`/`activating` → деактивирует,
иначе активирует. Это обработчик, навешиваемый по умолчанию.

#### `activate(el)`

Принудительно включает (с учётом guard'а состояния и `allow_interrupt`).

#### `deactivate(el)`

Принудительно выключает (с учётом guard'а состояния и `allow_interrupt`).

```js
tg.toggle('#item');
tg.activate(document.querySelector('[trigger]'));
tg.deactivate('#accordion [trigger]');   // на все совпадения
```

#### `clone(params)`

Создаёт новый экземпляр `Toggle` на основе текущего: параметры и хуки исходного экземпляра берутся
за основу, переданный `params` их переопределяет, а в новый экземпляр добавляется ссылка на
оригинал через параметр `parent`.

В отличие от `Modal.clone()`, DOM не клонируется — `Toggle` не владеет собственными элементами,
поэтому обычно при клонировании переопределяют `trigger` (и при необходимости `target`), чтобы
тот же конфиг применить к другим элементам без повторной привязки к исходным.

```js
const base = new Toggle({ duration: 0.3, on_activate() { /* … */ } });

// та же логика, но на другом наборе триггеров
const copy = base.clone({ trigger: '#sidebar [trigger]' });
```

## 7. Статические методы

### `Toggle.find(element | selector)`

Возвращает экземпляр `Toggle`, связанный с переданным **триггером** (или `undefined`).

```js
const instance = Toggle.find(document.querySelector('[trigger]'));
instance?.deactivate(someTrigger);
```

## 8. Разрешение целей (target)

Цели триггера определяются при **каждом** переключении в таком порядке:

1. `params.target` — если задан:
   - **функция** → вызывается как `target(trigger)`; может вернуть селектор, `Element`,
     `NodeList` или массив;
   - **селектор / Element / NodeList / Array** → используется как есть.
2. Иначе — атрибут `target=""` самого триггера (CSS-селектор).
3. Если ничего не найдено — целей нет, триггер переключает только себя.

Результат приводится к массиву элементов хелпером `elements()` из трейта (без побочных эффектов:
несовпавший селектор → пустой список).

```html
<div trigger target="#panel">Кнопка</div>
<div id="panel" content>…</div>
```

```js
// Цель как замыкание (например, относительно триггера):
new Toggle({ target: trigger => trigger.parentElement.querySelector('.drop') });
```

## 9. CSS-анимация через атрибут state

Единственный механизм визуала — CSS по атрибуту `[state]`. Класс не задаёт `transition`
самостоятельно — это делает разработчик.

```css
/* Спойлер: видим в active/activating, скрыт в inactive/deactivating */
[content] { overflow: hidden; transition: height .3s, opacity .3s; }

[content][state="inactive"],
[content][state="deactivating"] { opacity: 0; height: 0; }

[content][state="active"],
[content][state="activating"]   { opacity: 1; }
```

> **Совет:** разделяя `activating`/`deactivating`, можно задать разные анимации входа и выхода.

## 10. Примеры

### Спойлер по атрибуту `target`

```html
<button trigger target="#more">Подробнее</button>
<div id="more" content>Скрытый текст…</div>
```

```js
import Toggle from './toggle.js';
new Toggle({ duration: 0.3 });   // trigger по умолчанию '[trigger]'
```

### Аккордеон (взаимоисключение вручную)

```js
const accordion = new Toggle({
    duration: 0.3,
    trigger: '#accordion [trigger]',
    on_activate(trigger) {
        // групп «из коробки» нет — закрываем соседей сами
        for (const other of document.querySelectorAll('#accordion [trigger]'))
            if (other !== trigger) accordion.deactivate(other);
    },
});
```

### Кнопка-переключатель (без целей)

```js
new Toggle({
    trigger: '.switch',
    target: null,                         // переключает только сам триггер
    duration: 0.25,
    on_activate:   () => console.log('on'),
    on_deactivate: () => console.log('off'),
});
```

### Hover-меню (target как замыкание)

В режиме `'hover'` вешайте триггер на **обёртку**, которая DOM-содержит и подпись, и выпадающий
список — тогда наведение на пункты считается «внутри» (`mouseleave` не срабатывает при переходе
на потомка). Триггер на саму подпись закроет меню при наведении на пункты. См. раздел [11](#11-известные-особенности-и-ограничения).

```html
<div class="menu" trigger>
    <div class="menu-trigger">Меню ▾</div>
    <div class="menu-drop"><!-- пункты --></div>
</div>
```

```js
new Toggle({
    action: 'hover',
    duration: 0.2,
    trigger: '.menu',
    target: trigger => trigger.querySelector('.menu-drop'),
});
```

> **Зазор закрывает меню.** Не оставляйте пустого промежутка между подписью и выпадашкой
> (`margin-top` и т.п.) — курсор пройдёт по «дыре» над `body`, сработает `mouseleave`, и меню
> закроется. Используйте смежные элементы (`top: 100%` без отступа) или соедините их `padding`.

### Программное управление и `find`

```js
const tg = new Toggle({ trigger: '[trigger]' });

tg.activate('#item');
console.log(document.querySelector('#item').getAttribute('state'));   // 'activating' → позже 'active'

const inst = Toggle.find(document.querySelector('[trigger]'));
inst.toggle(someTrigger);
```

## 11. Известные особенности и ограничения

### Триггеры привязываются один раз

Селектор `params.trigger` резолвится в конструкторе. Элементы, добавленные в DOM позже,
обработчик не получают — добавляйте их вручную или используйте делегирование.

### Цели разрешаются при каждом переключении

В отличие от триггеров, цели вычисляются на каждый вызов — это позволяет `target`-замыканию
учитывать текущий DOM и работать с динамическими элементами.

### Групп «из коробки» нет

Взаимоисключение (аккордеон) реализуется вручную через `deactivate()` соседей в хуке.

### Состояния триггера и целей независимы

Один target, управляемый несколькими триггерами, может рассинхронизироваться с их состояниями —
это сделано намеренно (для комбинирования цепочек toggle-ров).

### hover: учитываются DOM-потомки триггера, а не геометрия

`action: 'hover'` использует `mouseenter`/`mouseleave`. Они **не** срабатывают при переходе курсора
на DOM-потомка триггера (даже если тот спозиционирован вне его рамки). Поэтому для hover-меню вешайте
триггер на обёртку, содержащую и подпись, и выпадашку, и не оставляйте между ними геометрического
зазора (см. пример выше). Если цель — *сосед* триггера, наведение на неё закроет меню.

### Класс не управляет display

После выключения цель остаётся в DOM со `state="inactive"`. Если нужно `display: none` —
задайте это в CSS через `[state="inactive"]`.

## 12. Якорные ссылки (deep-linking)

При `anchor: true` активный триггер синхронизируется с хешем адресной строки: активация
дописывает `#id`, открытие страницы с `#id` сразу активирует соответствующий триггер. По умолчанию
`anchor` выключен (`false`); `keep_history: true` включает его принудительно.

### Какой id попадает в URL (авто-приоритет)

Для каждого триггера id вычисляется так:

1. если у **самого триггера** есть `id` — пишется он;
2. иначе, если у триггера ровно **одна** цель и у неё есть `id` — пишется `id` цели;
3. иначе — триггер в URL не отражается.

При коллизии (есть `id` и у триггера, и у единственной цели) **выигрывает id триггера**. Триггеры
без подходящего `id` (например, hover-меню без `id` у дропдауна) ничего не пишут в URL.

```html
<!-- id у цели → в URL попадёт #a2 -->
<div trigger target="#a2">Раздел 2</div>
<div id="a2" content>…</div>

<!-- id у самого триггера → в URL попадёт #theme-switch -->
<div class="switch" trigger id="theme-switch"></div>
```

### Поведение кнопки «назад» (`keep_history`)

URL всегда обновляется через `history.*` (никогда через `location.hash`) — поэтому **прокрутки к
элементу не происходит**, а история не засоряется при `keep_history: false`.

| `keep_history` | Метод | Кнопка «назад» |
|---|---|---|
| `false` (по умолчанию) | `replaceState` | уходит со страницы, переключения табов игнорируются |
| `true` | `pushState` + `hashchange` | возвращает предыдущий таб; пустой хеш закрывает текущий |

```js
// Табы с навигацией «назад/вперёд» между ними
new Toggle({ trigger: '#tabs [trigger]', keep_history: true });
```

### Прокрутка к элементу

Наш код прокрутку **не** вызывает. Если она нужна — сделайте триггер настоящей ссылкой
`<a href="#id">`: браузер сам проскроллит к элементу (это нативное поведение, независимое от
`Toggle`). Если прокрутка не нужна — используйте любой не-`<a>` элемент.

### Отключение

`anchor: false` полностью выключает работу с URL — поведение как до появления фичи.
