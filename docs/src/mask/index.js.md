# index.js

## Описание

Маски ввода для `<input>`. Заменяет устаревший `st_mask`.

Всё в одном классе: статические методы (`Mask.compile`, `Mask.build`, `Mask.run`, `Mask.run_all`, `Mask.render`, `Mask.caret_for`) — чистый движок без DOM, тестируется в голом Node (`node scripts/mask.test.mjs`); экземпляр — DOM-адаптер поверх него.

Все виды правок — набор, вставка (Ctrl+V), удаление, drag-and-drop, автозаполнение браузера, программный `set()` — проходят через один общий путь: правка применяется к каноническому «сырому потоку», поток прогоняется через движок, поле получает новый текст и корректную позицию каретки.

При `placeholder: 'always'` и фиксированной маске (без вариативной длины) включается **режим обособленных частей**: каждый слот — самостоятельная ячейка. Клик в ММ печатает в ММ, не сдвигая ГГГГ; Backspace/Delete очищают только свою ячейку. Пропущенные позиции живут в потоке «дырками» — символами-заполнителями (поэтому заполнитель не должен быть валидным символом слота).

## Конструктор

```js
new Mask(params)
```

| Параметр | По умолчанию | Описание |
|---|---|---|
| `target` | `'input[mask]'` | Селектор \| `Element` \| `NodeList` \| массив |
| `mask` | `null` | Маска (формы — ниже); `null` → берётся из атрибута `mask=""` самого поля |
| `filler` | `'_'` | Заполнитель; строка длиной с маску — позаполнительно (`'ДДММГГГГ'`) |
| `placeholder` | `true` | `true` — только введённое + авто-разделители (ведущие литералы вроде `'+7 ('` появляются при фокусе); `'always'` — полный скелет с заполнителями; `false` — без плейсхолдера |
| `flow` | `true` | `false` — блоки изолированы (режим обособленных частей): набор не перетекает в следующий блок (переход — набором его разделителя или кликом), Backspace/Delete не выходят за границы блока |
| `rewrite` | `false` | `true` — ввод всегда начинается с начала блока: клик в середину блока и набор перезаписывают блок целиком; когда блок заполнен, следующий символ затирает его и пишется первым. Работает в режиме обособленных частей |
| `caret` | `true` | `true` — обычное поведение каретки; `false` — каретка прижата к концу (в режиме обособленных частей — к концу блока, по которому кликнули); `'hide'` — то же плюс каретка скрыта через `caret-color: transparent !important` (инлайн с `!important` — самый высокий приоритет в каскаде, обычным CSS не вернуть) |
| `coerce_type` | `true` | Приводить `type="number"`/`"email"` к `type="text"` + `inputmode` (эти типы не отдают selection API). `false` — не трогать тип (будет `console.warn`) |
| `max_raw` | `null` | Лимит длины `raw` (пользовательских символов). Оверрайдит атрибут `maxlength` поля |
| `validate` | `null` | `true` — прокидывать `complete` в нативный `setCustomValidity` для всех полей; `null` — только для приведённых `number`/`email`; `false` — не трогать нативную валидность |
| `validation_message` | `null` | Сообщение для `setCustomValidity` при неполном значении: строка или `(input, state) => string`. По умолчанию — RU-текст по типу поля |

### Коллбэки

Все коллбэки одинаковы по форме: `input` первым аргументом, полезная нагрузка объектом; `false` отменяет, возвращённая строка заменяет.

| Коллбэк | Когда | Особенности |
|---|---|---|
| `before_init(params)` / `on_init(params)` | Инициализация | |
| `before_char(input, { char })` | На каждый входной символ до разбора | Нормализация: регистр, транслитерация. Вернуть строку — заменить, `false` или `''` — выбросить символ. Должен быть идемпотентным (поток прогоняется заново на каждой правке) |
| `before_slot(input, { char, slot, cells, raw, node })` | Символ подошёл слоту и вот-вот в него встанет | Автодополнение (вернуть строку: `'3'` → `'03'`), автоисправление (`false` — отклонить). `cells` — уже заполненные ячейки по номерам слотов; в отличие от `raw`, индексы не сбиваются пропущенными позициями |
| `before_input(input, { type, insert, start, end })` | Перед каждой правкой | `return false` отменяет |
| `on_input(input, state)` | После каждой правки | |
| `before_paste(input, text)` | Перед вставкой | `false` — отменить, строка — заменить вставляемый текст |
| `on_paste(input, state)` | После вставки | |
| `on_accept(input, state)` | `raw` реально изменился | |
| `on_complete(input, state)` | Маска стала заполненной | |
| `on_incomplete(input, state)` | Перестала быть заполненной | |
| `on_mask_change(input, state)` | Переключилась активная маска | `state.mask_id` — ключ словаря или индекс массива |

`state` = `{ raw, formatted, complete, mask_id, progress }`.

`before_char` и `before_slot` — обычные коллбэки: привязаны к `this`, переносятся в `clone()` и переопределяются на уровне отдельной маски (`{ pattern, before_slot }`). Разделены они не по прихоти, а потому что срабатывают в разные моменты: `before_char` видит любой символ, включая мусор при вставке, и не знает позиции; `before_slot` вызывается только когда символ уже подошёл слоту, и знает его номер и соседние ячейки. Объединять их в один хук нельзя — тогда правило вида «цифра больше 3 → дописать 0» применялось бы и к мусору при вставке (`'т' > '3'` истинно для строк) и портило бы значение.

## Формы параметра `mask`

| Форма | Смысл |
|---|---|
| `'строка'` | Позиционный шаблон (синтаксис ниже) |
| `RegExp` | Фильтр-режим: посимвольный allowlist — для email/URL, где позиции не фиксированы. Внутри это обычный повторитель `{слот;0-}` |
| `{ pattern, filler?, before_char?, before_slot?, valid? }` | Шаблон с пер-масочными настройками; `valid` — RegExp по всему значению, дополняет `complete` (например, диапазон октета IP) |
| `{ filter, valid?, max_length?, before_char? }` | Фильтр-режим |
| `{ numeral: {…} }` | Числовой форматтер (деньги/суммы): заполнение справа, группировка, дробная часть — см. ниже |
| `функция(ctx)` | Возвращает любую из форм; вызывается на каждой правке. `ctx = { stream, raw, value, input }`, где `stream` — набираемое значение (по нему собирают шаблон, зависящий от длины: разряды по три) |
| `[…]` | Массив любых форм — динамический выбор: выигрывает маска, принявшая больше символов; при равенстве — заполненная; затем первая (ИНН 10/12, телефон 8/+7) |
| `{ ru: '…', by: '…' }` | Словарь именованных масок: по первым символам подтягивается самая подходящая, ключ приходит в `state.mask_id`, атрибут `mask_id` и событие `on_mask_change` |

## Синтаксис шаблона

| Синтаксис | Значение |
|---|---|
| `0` | Обязательная цифра |
| `a` | Обязательная буква (юникод, кириллица включена) |
| `*` | Любой печатный символ |
| `\x` | Экранированный литерал: `\0`, `\a`, `\*`, `\[`, `\{`, `\\` |
| прочие символы | Статические разделители |
| `{{re}}` | Один символ по регулярному выражению: `{{[0-9a-f]}}` |
| `[ … ]` | Необязательный блок — появляется, когда пользователь начал его вводить: `00.00.0000[ 00:00]` |
| `{unit;N}` `{unit;min-max}` `{min-}` `{-max}` | Повторитель; `unit` — под-шаблон: `{0;1-3}` — 1–3 цифры, `{0;1-}` — до бесконечности, `{{{[0-9a-f]}};4}` — 4 hex-символа |
| `{c=>s}` | Трансформация: ввод `c` даёт `s`; вставка текста, начинающегося с `s`, тоже матчится: `{8=>+7}` |

Ограничение: несбалансированные `{` `}` внутри `{{re}}` нужно экранировать (`\{`). Некорректный блок не роняет компиляцию — превращается в литералы с `console.warn`.

## Числовой режим (`{ numeral }`)

Форма для денег/сумм. Позиционный шаблон тут не подходит: заполнение идёт **справа** —
набранные цифры это счётчик младших разрядов (при `fraction: 2` первый `1` даёт `0,01`,
следующая цифра сдвигает разряды влево), целая часть группируется.

| Опция | По умолчанию | Смысл |
|---|---|---|
| `fraction` | `0` | Знаков дробной части (заполняется справа) |
| `group` | `' '` | Разделитель тысяч (`''` — выключить) |
| `decimal` | `','` | Десятичный разделитель |
| `prefix` / `suffix` | `''` | Литералы до/после числа (`suffix: ' ₽'`) |
| `sign` | `false` | Разрешить ведущий `-` |
| `min` / `max` | — | Диапазон значения (входит в `complete`) |
| `message` | — | Сообщение нативной валидации при значении вне диапазона |

```js
new Mask({ target: 'input[name=sum]', caret: 'hide',
    mask: { numeral: { fraction: 2, group: ' ', decimal: ',', suffix: ' ₽', min: 0 } } });
```

`raw` содержит только значащие цифры (и знак): `12 345,67 ₽` → `raw` = `'1234567'`. Ввод —
end-append, поэтому естественно сочетается с `caret: 'hide'`. **`set()`**: голый ряд цифр
трактуется как заполнение справа (`set('1')` → `0,01`), а строка с десятичным разделителем —
как реальное число с округлением до `fraction` (`set('1234.5')` → `1 234,50`).

Пустой `type="number"` без явной маски получает numeral по умолчанию (`fraction: 0` —
сгруппированное целое) и ввод только цифр (см. `coerce_type`).

## Публичные методы

| Метод | Описание |
|---|---|
| `raw(target?)` | Чистое значение поля (только пользовательские символы) |
| `state(target?)` | `{ raw, formatted, complete, mask_id, progress }` |
| `set(target?, value)` | Программная установка (raw или отформатированное — мусор отфильтруется) |
| `clear(target?)` | Очистить поле (по умолчанию — все) |
| `set_mask(mask, target?)` | Заменить маску на лету (по умолчанию — всем полям) |
| `clone(params)` | Копия экземпляра с переопределёнными параметрами |
| `get value` | Массив отформатированных значений всех привязанных полей |
| `Mask.find(el)` | Экземпляр, владеющий элементом (`hasInstanceSymbol`) |
| `Mask.compile / build / run / run_all / render / caret_for` | Статический движок — работает без DOM, можно использовать для форматирования строк вне полей |

`target` необязателен, когда маска привязана ровно к одному полю; при нескольких — понятная ошибка вместо угадывания. `set` при единственном поле принимает одно значение: `mask.set('12.03.2026')`.

### API на самом элементе

Каждое привязанное поле получает методы владельца — можно работать с полем, не доставая экземпляр:

```js
input.raw();                     // '9123456789'
input.state();                   // { raw, formatted, complete, mask_id, progress }
input.set('89123456789');        // → +7 (912) 345-67-89
input.clear();
input.set_mask('0000 0000 0000 0000');
```

Свойства неперечислимые (не попадают в `Object.keys` и `JSON.stringify`); занятое имя — нативное или чужое — не затирается, вместо этого пишется предупреждение. То же самое умеют `Select` (`select.open()`, `close()`, `toggle()`) и `Toggle` (`trigger.toggle()`, `activate()`, `deactivate()`) — общий механизм `expose` из `_traits/hasInstanceSymbol.js`.

## Поведение каретки и правок

- Разделители проставляются сами; каретка перескакивает их при наборе.
- Backspace удаляет предыдущий пользовательский символ, перескакивая разделители; трансформация (`+7`) удаляется атомарно.
- Delete удаляет только следующий пользовательский символ.
- `placeholder: 'always'` + фиксированная маска: печать поверх ячеек, части независимы (см. Описание); каретка за последней ячейкой печатает в первую пустую; Delete/Backspace пропускают пустые ячейки и очищают ближайшую занятую.
- `flow: false` запирает набор и удаление в блоке каретки (блок — непрерывный ряд ячеек между разделителями).
- `caret: false` / `'hide'`: каретка прижата к концу (в режиме обособленных частей — к концу того блока, по которому кликнули). Раз в середину нельзя писать, из неё нельзя и удалять: любое частичное выделение (двойной клик по слову или блоку, протяжка) расширяется до выделения всего значения, а сама правка принудительно приводится к прижатой позиции — это закрывает и обходные пути вроде перетаскивания текста мышью. Остаются ровно два действия: дописывать/стирать с конца или заменить значение целиком.
- Набор разделителя переводит каретку в следующий блок из любой позиции, в том числе из наполовину заполненного (`1.3.2026` → `1Д.3М.2026`).
- `rewrite: true`: набор `12312` в блоке ДД даёт `12` → `3` (блок был полон — затёрся) → `31` → `2`. Символ, не подходящий слоту, блок не затирает.
- Правка в середине строки переформатирует хвост, каретка остаётся на месте.
- Вставка любого текста: мусор отфильтровывается («тел: 8 (912) 345-67-89» → телефон), работает в середину и поверх выделения.
- Автозаполнение браузера и undo/redo сверяются страховочным путём по событию `input`.

## Атрибуты поля после каждой правки

| Поле | Описание |
|---|---|
| `mask_id` | Индекс активной маски (для массивов) или ключ (для словаря) |
| `progress` | Число принятых пользовательских символов |
| `is_complete` | `"true"` если маска заполнена |

Атрибуты обычные, не `data-*` — селекторы вида `input[is_complete="true"]` работают напрямую.

## Нативная валидация (Constraint Validation API)

Признак `complete` прокидывается в `input.setCustomValidity(complete ? '' : message)`, поэтому
работают штатные `:invalid`, `input.reportValidity()` и блокировка отправки формы — без
самописной проверки. Включена автоматически для приведённых `number`/`email`; для остальных
типов — параметром `validate: true`. Пустое поле считается валидным (за обязательность отвечает
нативный `required`). Сообщение задаётся `validation_message` (или пер-масочным `{ message }`).

## `type="number"` / `type="email"`

Эти типы не отдают selection API (spec: выделение есть только у `text/search/tel/url/password`),
поэтому при `coerce_type: true` (по умолчанию) поле приводится к `type="text"` + подходящему
`inputmode`. `number` дополнительно принимает только цифры (независимо от токенов маски), а без
явной маски получает numeral по умолчанию; `email` без явной маски — универсальную фильтр-маску.

**Caveat:** приведение типа меняет отправляемое значение формы — для numeral с группировкой
`input.value` = `'1 234 567 ₽'`, число берётся через `input.raw()` (или зеркалируется в скрытое поле).

## `maxlength`

Атрибут `maxlength` снимается с поля (нативный считает символы **отформатированного** значения
и воевал бы с движком) и применяется как лимит длины **`raw`** — пользовательских символов.
Программный оверрайд — параметр `max_raw`.

## Ограничения (осознанные)

- Undo/redo не поддерживается: программные записи значения опустошают нативную историю; значение просто приводится к маске.

## Пример

```js
new Mask({
    target: 'input[name=phone]',
    mask: [
        '{8=>+7} (000) 000-00-00[ доб. {0;1-5}]',
        '+7 (000) 000-00-00[ доб. {0;1-5}]'
    ],
    on_complete: (input, state) => console.log('готово:', state.raw)
});

new Mask({
    target: 'input[name=date]',
    mask: '00.00.0000',
    filler: 'ДДММГГГГ',
    placeholder: 'always',
    before_slot: (input, { char, slot }) => {
        if (slot == 0 && char > '3') return '0' + char;   // день: 4 → 04
        if (slot == 2 && char > '1') return '0' + char;   // месяц: 3 → 03
    }
});
```

### Автодополнение и автоисправление (`before_slot`)

Это два разных правила, оба через один хук:

- **Автодополнение** — когда цифра однозначна и не может быть началом двузначного числа. День: `4` не может начинать `40+`, значит это `04`. Месяц: `2`–`9` → `02`–`09`. Цифры `1`–`3` для дня неоднозначны (`1` → `01`, `10`–`19`, `31`), поэтому дополнять их нельзя — ждём второй разряд.
- **Автоисправление** — ограничение второго разряда, чтобы невозможными стали `00`, `13`–`19` (месяц) и `32`–`39` (день). Именно оно, а не автодополнение, не даёт указать месяц больше 12.

```js
before_slot: (input, { char, slot, cells }) => {
    if (slot == 0 && char > '3') return '0' + char;                    // день:   4 → 04
    if (slot == 1 && (cells[0] == '3' && char > '1'
                   || cells[0] == '0' && char == '0')) return false;   // не 32-39 и не 00
    if (slot == 2 && char > '1') return '0' + char;                    // месяц:  3 → 03
    if (slot == 3 && (cells[2] == '1' && char > '2'
                   || cells[2] == '0' && char == '0')) return false;   // не 13-19 и не 00
}
```

Нормализация символов — второй хук, `before_char`. Он не знает позиции, зато видит любой символ:

```js
// Госномер: латиница с клавиатуры превращается в кириллицу
before_char: (input, { char }) => {
    char = char.toUpperCase();
    return lat2cyr[char] ?? char;
}
```

### Переключение маски из события

Событий и методов достаточно, чтобы разбирать ввод посимвольно и самому выбирать подмаску: `before_input` отдаёт вставляемый текст до применения правки, `set_mask` меняет маску поля на лету.

```js
const doc = new Mask({
    target: 'input[name=doc]',
    mask: '{*;0-}',
    before_input: (input, edit) => {
        if (!edit.insert || doc.raw(input)) return;     // только первый символ

        let kind = Object.keys(DOC_MASKS).find(k => DOC_MASKS[k].test(edit.insert[0]));
        if (!kind) return false;                        // символ не подошёл ни одной маске

        doc.set_mask(DOC_MASKS[kind].mask, input);      // дальше правка идёт по новой маске
    }
});
```

Тот же приём с внешним источником — `<select>` со странами:

```js
const phone = new Mask({ target: 'input[name=phone]', mask: COUNTRY_MASKS.ru });

select.addEventListener('change', event =>
    phone.set_mask(COUNTRY_MASKS[event.target.value], 'input[name=phone]'));
```

Если выбор маски выражается правилом «по первым символам», словарь делает это сам, без событий — см. форму `{ ru: …, by: … }` выше.
