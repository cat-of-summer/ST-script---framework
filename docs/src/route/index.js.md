## 1. Описание

`Route` — клиентский диспетчер **реакций**. В CMS и кастомных проектах один JS подключается на
множестве страниц; `Route` по **текущему URL** решает, какие замыкания выполнить. Это аналог
серверного `Route.php`, но в браузере «запрос» — это всегда уже загруженный документ (метод
всегда `GET`).

**Ключевые принципы:**

- **Только реакция, без навигации.** Класс не трогает `history`, не подменяет URL и не
  перехватывает клики/формы. Он лишь сопоставляет `location.pathname` с маршрутами и вызывает
  замыкания.
- **Строго замыкания.** И обработчики, и middleware — функции (не имена классов/строки).
- **Старт на `DOMContentLoaded`.** К этому моменту DOM построен; `Modal`/`Toggle` уже
  инициализированы (если созданы синхронно). `App` (`st-app`) может ещё дорендериваться — см.
  раздел [12](#12-известные-особенности-и-ограничения).
- **События — только DOM.** Диспатчатся `CustomEvent` на `document` (без отдельной шины): один
  «предвестник» до сопоставления и события на каждый совпавший маршрут — чтобы любой код
  (Modal/App/Toggle/свой) мог реагировать по имени события. См. раздел [9](#9-события).
- **Нет зависимостей.** Чистый ES6-класс, экспортируется как ES-модуль (`export default`).

## 2. Модель: реакции на текущий URL

```
new Route({ point, name, match_all })
        │  регистрация (синхронно)        диспетчеризация (на DOMContentLoaded)
        ├── get('index.html', fn)         dispatch → document 'route' (предвестник, до матчинга)
        ├── get('catalog.html', fn)       location.pathname ─► сопоставление с #routes
        ├── get('product-{id}.html', fn, 'product')          ─► для каждого совпавшего:
        └── get('*', fn)  ← глобальная        middleware-гарды → обработчик →
                                              document 'route::<alias>' / 'route::<uri>'
```

Маршруты регистрируются синхронно при загрузке скрипта. Сопоставление и вызовы происходят один
раз — после `DOMContentLoaded`. Если DOM уже готов на момент создания, диспетчеризация
откладывается микротаской — чтобы синхронно объявленные ниже маршруты успели зарегистрироваться.

## 3. Параметры конструктора

```js
new Route(params)
```

Все параметры необязательны.

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `point` | `string` | `'/'` | Базовый префикс пути. Маршруты строятся относительно него — удобно, если сайт лежит в подкаталоге. Входит в итоговый паттерн (как `point()` в PHP). |
| `name` | `string` | `''` | Неймспейс событий. Предвестник — `route:<name>`, совпадения — `route:<name>:<alias>` / `route:<name>:<uri>`. Нужен, если на странице **несколько роутеров** (см. раздел [9](#9-события)). По умолчанию пусто — имена сводятся к `route` (предвестник) и `route::<alias>` / `route::<uri>` (совпадения). |
| `match_all` | `boolean` | `true` | `true` — выполнять замыкания **всех** совпавших маршрутов (например, глобальный `*` + страничный); `false` — только **первого** совпавшего. |
| `strict_mode` | `boolean` | `true` | Нормализация путей при построении паттерна (паритет с PHP). На само сопоставление влияет слабо: хвостовой слеш всегда необязателен. Можно переопределить для отдельного маршрута через 3-й аргумент `get`. |

## 4. Хуки жизненного цикла

Хуки инициализации (как в `Modal`/`Toggle`). Привязываются к экземпляру через `.bind(this)` —
внутри хука `this` указывает на экземпляр `Route`. Для реакции на загрузку и совпадения маршрутов
используйте **события** (раздел [9](#9-события)) и сами замыкания обработчиков.

| Хук | Когда | Аргумент |
|---|---|---|
| `before_init` | до настройки экземпляра | `params` |
| `on_init` | после инициализации (до `DOMContentLoaded`) | `params` |

## 5. Регистрация маршрутов

### `get(uri, closure, params = {})`

Регистрирует маршрут. `uri` строится относительно `point` и текущего префикса. `closure` —
обработчик-замыкание. Возвращает `this` (цепочки).

Третий аргумент — **строка-алиас** или объект параметров:

```js
router.get('product-{id}.html', fn, 'product');          // алиас → событие route::product
router.get('product-{id}.html', fn, { alias: 'product', strict_mode: false });
```

Повторная регистрация **того же итогового пути** бросает ошибку (как в PHP). Единственный метод
регистрации — `get`: на загрузке страницы запрос всегда `GET`, других глаголов нет.

## 6. Синтаксис путей: `{param}` и wildcard

| Синтаксис | RegExp | Что захватывает |
|---|---|---|
| `{name}` | `([^/]+)` | один сегмент (без `/`); попадает в `this.params.name` и в позиционные аргументы |
| `*` | `(.*)` | весь остаток пути (может содержать `/`); попадает в `this.params['*']` и последним позиционным аргументом |
| литералы | экранируются | совпадают как есть |

Хвостовой слеш необязателен: паттерн `about` совпадёт и с `/about`, и с `/about/`. Значения
сегментов декодируются через `decodeURIComponent` (битый `%`-escape не роняет матч — возвращается
как есть).

```js
router.get('catalog/{category}/{id}', (category, id) => { /* … */ });
router.get('docs/*', (rest) => { /* docs/a/b/c → rest = "a/b/c" */ });
```

## 7. Обработчик и middleware

### Аргументы обработчика — **позиционные**

Значения `{param}` передаются по порядку появления; wildcard-остаток — последним. `this`
привязан к экземпляру `Route`, поэтому помимо аргументов доступны контекстные свойства
(`this.params`, `this.query`, `this.path`, `this.url`).

```js
router.get('product-{id}.html', function (id) {
  console.log(id, this.query, this.path);   // '42', {...}, '/.../product-42.html'
});
```

### `middleware(fn | fn[])`

Добавляет гард(ы)-замыкания в **текущий контекст** (см. раздел [8](#8-группы-префиксы-точка-входа)).
Гард получает **те же позиционные аргументы**, что и обработчик (и `this`). Если хотя бы один
вернёт falsy — маршрут пропускается. Возвращает `this`.

```js
const allowed = ['1', '2'];
router
  .middleware((id) => allowed.includes(id))   // гард для следующего маршрута в этом контексте
  .get('product-{id}.html', (id) => { /* выполнится только для id 1 и 2 */ });
```

> `middleware` действует на **область** (текущий кадр/группу), а не на одну строку. Чтобы
> ограничить гард единственным маршрутом, оборачивайте его в группу (раздел [8](#8-группы-префиксы-точка-входа)).

## 8. Группы, префиксы, точка входа

### `prefix(uri)`

Открывает вложенный контекст с добавленным префиксом пути (наследует middleware родителя).
Возвращает `this`.

### `group(closure)`

Выполняет `closure()` в текущем контексте и затем закрывает его (снимает верхний кадр).

```js
router.prefix('admin').middleware(() => isAdmin).group(() => {
  router.get('dashboard', () => { /* /admin/dashboard, под гардом isAdmin */ });
  router.get('users/{id}', (id) => { /* /admin/users/{id} */ });
});

router.get('login', () => { /* вне группы — без гарда isAdmin */ });
```

> Это перенос статического `prefix()->group()` из PHP на стек контекста экземпляра.

## 9. События

`Route` диспатчит только **DOM `CustomEvent` на `document`** (отдельной шины `on`/`emit` нет —
слушайте через `document.addEventListener`). Имена неймспейсятся параметром `name` (раздел
[3](#3-параметры-конструктора)); ниже — формат и его вид при `name=''` (по умолчанию).

| Событие | Когда | `detail` |
|---|---|---|
| `route:<name>` → `route` | один раз, **до** сопоставления `uri` — предвестник скорого матчинга | `{ name, path, query, url }` |
| `route:<name>:<alias>` → `route::<alias>` | на каждый совпавший маршрут, если задан алиас | объект маршрута |
| `route:<name>:<uri>` → `route::<uri>` | на каждый совпавший маршрут, по исходному `uri` | объект маршрута |

Маршрутные события используют **два** разделителя (`route:<name>:<…>`), поэтому при пустом имени это
`route::<alias>` / `route::<uri>`. Двойное двоеточие — намеренно: предвестник `route:<name>` никогда
не совпадёт с маршрутным `route:<name>:<…>`, а пустое имя не создаст коллизий, даже если алиас равен
имени другого роутера.

Объект маршрута в `detail` (у маршрутных событий):

```js
{ name, pattern, uri, alias, params, values, query, path, url }
//  ↑ неймспейс  ↑ полный путь  ↑ raw uri  ↑ {name:value}  ↑ позиционные значения
```

```js
// предвестник — один раз, до матчинга (удобно для инициализации виджетов)
document.addEventListener('route', () => initWidgets());

// совпадение по алиасу 'product' (имя пустое → двойное двоеточие)
document.addEventListener('route::product', (e) => track('product', e.detail.params.id));

// совпадение по uri
document.addEventListener('route::catalog.html', () => initCatalog());
```

### Несколько роутеров на странице

Инстансы независимы (свои таблицы маршрутов и контекст), но события идут на общий `document`.
Чтобы развязать их, задайте каждому свой `name` — тогда имена не пересекаются:

```js
new Route({ name: 'shop' }).get('product-{id}.html', fn, 'product');  // route:shop  + route:shop:product
new Route({ name: 'blog' }).get('post-{id}.html',    fn, 'post');     // route:blog  + route:blog:post

document.addEventListener('route:shop:product', …);   // только товары
document.addEventListener('route:blog:post', …);      // только посты
```

> Без `name` оба роутера дадут одинаковые имена (`route`, `route::product`…) — слушатель не отличит
> источник. `detail.name` тоже доступен для фильтрации.

## 10. Публичные свойства и методы

### Геттеры / свойства

| Член | Описание |
|---|---|
| `routes` | Массив зарегистрированных маршрутов (только чтение). |
| `params` | Извлечённые параметры текущего совпадения (`{ name: value, '*': rest }`). |
| `query` | GET-параметры текущего URL (объект из `location.search`). |
| `path` | `location.pathname` на момент диспетчеризации. |
| `url` | `location.href` на момент диспетчеризации. |

### Методы

| Метод | Назначение |
|---|---|
| `get(uri, closure, alias \| params?)` | Зарегистрировать маршрут (3-й аргумент — строка-алиас или объект). → `this` |
| `prefix(uri)` | Открыть вложенный контекст с префиксом. → `this` |
| `group(closure)` | Выполнить замыкание в контексте и закрыть его. |
| `middleware(fn \| fn[])` | Добавить гард(ы) в текущий контекст. → `this` |
| `clone(params)` | Новый `Route` на базе текущего (слияние параметров/хуков, переопределение через `params`, ссылка на оригинал в `parent`). |

## 11. Примеры

Полная рабочая демонстрация (несколько страниц + общий `routes.js`) — в
[`examples/route/`](../../../examples/route/).

### Реакции по страницам + глобальная реакция

```js
const router = new Route({ match_all: true });

router.get('*', function () {           // на каждой странице
  highlightNav(this.path);
}, 'global');
router.get('index.html', () => initHome(), 'home');
router.get('catalog.html', function () {
  filterCatalog(this.query.category);   // ?category= из URL
}, 'catalog');
```

### Параметр пути + гард + алиас

```js
const allowed = ['1', '2'];
router
  .middleware((id) => allowed.includes(id))
  .get('product-{id}.html', function (id) {
    document.getElementById('title').textContent = `Товар #${id}`;
  }, 'product');

// внешний код реагирует на совпадение этого маршрута (имя пустое → двойное двоеточие)
document.addEventListener('route::product', (e) => track('view', e.detail.params.id));
```

### Группа с префиксом и middleware

```js
router.prefix('account').middleware(() => isLoggedIn).group(() => {
  router.get('profile', () => renderProfile());
  router.get('orders/{id}', (id) => renderOrder(id));
});
```

### Координация с другими механизмами через события

```js
// Любой модуль слушает предвестник матчинга (до сопоставления uri) без ссылки на router:
document.addEventListener('route', () => {
  new Modal({ trigger: '[data-modal]' });
  new Toggle({ trigger: '#tabs [trigger]' });
});
```

## 12. Известные особенности и ограничения

### Старт один раз, на `DOMContentLoaded`

Сопоставление и вызовы происходят однократно. SPA-навигации (перехвата кликов, `pushState`) нет —
это диспетчер реакций для классических многостраничных проектов.

### `App` (`st-app`) рендерится асинхронно

На момент `DOMContentLoaded` кастомные элементы `App` могут ещё дорендериваться. Реакции, которым
нужна готовая App-разметка, слушайте per-instance событие `App` `'rendered'`.

### `point` входит в паттерн, не вычитается

Базовый префикс добавляется в итоговый путь маршрута и сопоставляется с полным `location.pathname`
(как в PHP). Для подкаталога задавайте `point` равным этому подкаталогу.

### `middleware` действует на область

Гард применяется ко всему текущему кадру/группе, а не к одной строке. Для гарда на единственный
маршрут используйте отдельную `prefix().group()`.

### Только `GET`

Других глаголов нет, формы/`fetch` не перехватываются — загрузка документа всегда `GET`.
